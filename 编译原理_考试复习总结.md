## 二、概论

### 2.1 编译程序基本概念

- **编译程序（Compiler）**：将高级语言程序翻译为低级（汇编/机器）语言程序。
- **解释程序（Interpreter）**：边读边执行（Python/JS），不产生独立目标程序。
- **翻译要求**：等价、高效。

### 2.2 编译程序的逻辑结构（各阶段）

```
源程序
  ↓  词法分析（Lexical Analyzer / Scanner）
     → Token 序列
  ↓  语法分析（Parser）
     → 语法树
  ↓  语义分析（Semantic Analyzer）
     → 带语义信息的语法树
  ↓  中间代码生成（Intermediate Code Generator）
     → 四元式 / 逆波兰 / LLVM IR
  ↓  代码优化（Code Optimizer）
  ↓  目标代码生成（Code Generator）
目标程序
```

贯穿全过程的辅助模块：**符号表管理** 和 **错误检查/处理**。

### 2.3 各阶段功能速记

| 阶段 | 输入 | 输出 | 核心任务 |
|------|------|------|---------|
| 词法分析 | 字符串 | Token 串 `(类别, 属性值)` | 识别单词，填符号表 |
| 语法分析 | Token 串 | 语法树 | 识别句子（句法结构） |
| 语义分析 | 语法树 | 带类型/作用域信息的树 | 类型检查、类型转换 |
| 中间代码生成 | 语义树 | 中间代码 | 机器无关表示 |
| 代码优化 | 中间代码 | 优化后中间代码 | 等价变换，提高效率 |
| 目标代码生成 | 中间代码 | 汇编/机器码 | 寄存器分配、指令选择 |

### 2.4 符号表

- 词法分析阶段进入符号表，后续阶段补充属性（类型、作用域、参数等）。

### 2.5 编译器实现技术（T形图）

- **自展技术**：用汇编写 C 子集编译器 → 用 C 子集写完整 C 编译器 → 自举编译。
- **交叉编译**：在 A 机上生成 B 机的目标代码。
- **自动生成工具**：Lex/Flex（词法）、YACC/Bison（语法）。

## 三、词法分析

### 3.1 单词种类

| 类别 | 例子 |
|------|------|
| 关键字/保留字 | `if`, `while`, `int` |
| 标识符 | `position`, `rate` |
| 常量 | `60`, `3.14`, `"str"` |
| 运算符 | `+`, `-`, `*`, `/`, `=` |
| 分隔符 | `(`, `)`, `;`, `,` |

### 3.2 正则表达式（Regular Expression）

**定义（递归）**：

| 正则式 | 对应正则集 |
|--------|----------|
| `ε` | `{ε}` |
| `∅` | `∅` |
| `a` | `{a}` |
| `(r)\|(s)` | `Lr ∪ Ls` |
| `(r)·(s)` | `Lr · Ls` |
| `(r)*` | `Lr*` |
| `(r)+` | `r·r*` |
| `(r)?` | `r\|ε` |
| `[abc]` | `a\|b\|c` |

**等价公理（常用）**：

```
r|s = s|r          r|r = r
r|ε = r             (r|s)|t = r|(s|t)
r(s|t) = rs|rt     r* = (ε|r)* = ε|rr*
r·ε = ε·r = r
```

**运算符优先级（高→低）**：`*` > `·`（连接）> `|`

### 3.3 有穷自动机（FA）

#### DFA（确定有穷自动机）

- 定义：`M = (K, Σ, f, S₀, Z)`
  - `K`：状态集；`Σ`：输入字母表；`f: K×Σ→K`（单值）；`S₀`：初态；`Z`：终态集
- 识别串 `x`：从初态出发，按 `x` 中字符逐步转移，最终到达**终态**则接受。
- `L(M)`：DFA M 识别的语言。

#### NFA（非确定有穷自动机）

- 定义：`f: K×Σ→℘(K)`（多值，到状态集合）
- NFA 识别有回溯，效率低 → 转换为等价 DFA 使用。
- **定理**：对任一 NFA N，存在等价 DFA M，`L(N) = L(M) = L(G)`（G 为正则文法）。

#### ε-NFA → NFA → DFA 转换

1. **Thompson法**（正则式 → NFA）：
   - `ε`、`a`：单个状态 + 单条边
   - `r₁|r₂`：引入新初态/终态，`ε` 边连接两子 NFA
   - `r₁·r₂`：将 r₁ 终态合并为 r₂ 初态（ε 边连接）
   - `r*`：引入新初态/终态，加 ε 自环和 ε 旁路

2. **子集法**（NFA → DFA）：
   - 将 NFA 的状态集合作为 DFA 的单个状态（`ε`-闭包）。

### 3.4 正则文法（3型文法）与 FA 的等价性

- **正则文法** ↔ **正则式** ↔ **有穷自动机**（三者等价）。
- 3型文法形如：`A → aB` 或 `A → a`（右线性）。

## 四、语法分析——自顶向下（LL(1)）

### 4.1 语法树 / 分析树

- 根为开始符号 S；内部结点为非终结符 VN；叶子为终结符 VT。
- **二义性文法**：同一句子对应多棵语法树。
- **最左推导**：每步替换最左边的非终结符。
- **最右推导（规范推导）**：每步替换最右边的非终结符，逆过程称**规范归约**。

### 4.2 自顶向下分析的问题

- **左递归**：`A → Aα`，自顶向下会死循环 → 必须消除。
- **回溯**：预测错产生式，需撤回 → 降低效率。

### 4.3 消除左递归

**直接左递归消除**：

```
A → Aα₁ | Aα₂ | … | Aαn | β₁ | β₂ | … | βm
```

改写为：

```
A  → β₁A' | β₂A' | … | βmA'
A' → α₁A' | α₂A' | … | αnA' | ε
```

**间接左递归**：先将产生式按顺序替换，消除代入后产生的直接左递归（矩阵方程法：`X = XA + B`，解为 `X = BA*`）。

### 4.4 FIRST 集

$$\text{FIRST}(\alpha) = \{a \mid \alpha \Rightarrow^* a\beta, \ a \in V_T\}$$

若 `α ⇒* ε`，则 `ε ∈ FIRST(α)`。

**计算规则**（对每个非终结符 X）：

1. `X ∈ VT`：`FIRST(X) = {X}`
2. `X → ε`：`ε ∈ FIRST(X)`
3. 对 `X → Y₁Y₂…Yk`：
   - 将 `FIRST(Y₁) - {ε}` 加入 FIRST(X)
   - 若 `ε ∈ FIRST(Y₁)`，再加入 `FIRST(Y₂) - {ε}`，以此类推
   - 若所有 `Yi` 均可推出 `ε`，则 `ε ∈ FIRST(X)`

### 4.5 FOLLOW 集

$$\text{FOLLOW}(X) = \{a \mid S\# \Rightarrow^* \ldots Xa \ldots, \ a \in V_T \cup \{\#\}\}$$

**计算规则**：

1. `# ∈ FOLLOW(S)` （S 为开始符号）
2. 对 `A → αXβ`：将 `FIRST(β) - {ε}` 加入 `FOLLOW(X)`
3. 对 `A → αXβ` 且 `ε ∈ FIRST(β)`，或 `A → αX`：将 `FOLLOW(A)` 加入 `FOLLOW(X)`
4. 反复迭代直到不变。

**注意**：`ε` 不会出现在 FOLLOW 集中，`#` 不会出现在 FIRST 集中。

### 4.6 LL(1) 文法定义

对每个产生式 `A → α₁ | α₂ | … | αm`，满足：

1. **FIRST 集不相交**：`FIRST(αi) ∩ FIRST(αj) = ∅`（`i ≠ j`）
2. **若某候选式可推出 ε**：`FIRST(αi) ∩ FOLLOW(A) = ∅`（对所有其他 i）

LL(1) 含义：**L**eft-to-right scan，**L**eftmost derivation，look-ahead **1** token。

### 4.7 非 LL(1) 文法改造

1. 消除左递归（见 4.3）
2. **提取左公因子**：
   ```
   A → αβ₁ | αβ₂ | … | αβn | γ
   ```
   改写为：
   ```
   A → αA' | γ
   A' → β₁ | β₂ | … | βn
   ```

### 4.8 递归下降分析法

- 每个非终结符对应一个递归函数。
- 查看当前输入符（lookahead），选择对应产生式右部。
- 示例（`E' → +TE' | ε`）：
  ```c
  expr_prime() {
      if (match(PLUS)) { advance(); term(); expr_prime(); }
      else if (match(';') || match(')'))  { /* ε */ }
      else error();
  }
  ```

### 4.9 预测分析（LL(1) 分析表）

**分析表 M[A, a]**：

- 若 `a ∈ FIRST(αi)`：`M[A, a] = "A→αi"`
- 若 `ε ∈ FIRST(αi)` 且 `a ∈ FOLLOW(A)`：`M[A, a] = "A→αi"`
- 其余：`M[A, a] = ERR`

**分析机工作过程**：

1. 初始栈：`#S`（S 在顶）
2. 栈顶为 VN：查表 M[栈顶, 当前输入]，弹出栈顶，逆序压入产生式右部
3. 栈顶为 VT：与当前输入匹配则同时弹出/移进；不匹配则报错
4. 栈和输入均为 `#`：分析成功

## 五、语法分析——自底向上（LR）

### 5.1 基本概念

- **自底向上**：从叶子向根归约（逆最右推导）。
- **两个核心动作**：
  - **移近（Shift）**：将当前输入符号压栈
  - **归约（Reduce）**：将栈顶匹配产生式右部的符号串替换为左部非终结符
- **句柄**：规范句型中可直接归约的子串（最左可归约子串）。
- **活前缀**：规范句型中不含句柄右部符号以右内容的前缀（即已移近但未完全归约的部分）。

### 5.2 LR 分析器结构

```
    输入: a₁ a₂ … aᵢ … aₙ #
               ↑
          ┌──────────────┐
          │   总控程序   │  ← 查分析表（ACTION, GOTO）
          └──────────────┘
               ↑↓
         分析栈（状态+符号）
```

- **ACTION[状态, 终结符]**：`sj`（移进到状态j）、`rk`（按第k产生式归约）、`acc`（接受）、`ERR`
- **GOTO[状态, 非终结符]**：归约后转移到的新状态

### 5.3 LR(0) 分析

#### LR(0) 项目

在产生式右部某位置加圆点（`·`），表示已识别程度：

- **移近项目**：圆点后为终结符，如 `A → α·aβ`
- **待约项目**：圆点后为非终结符，如 `A → α·Bβ`
- **归约项目**：圆点在最右，如 `A → α·`

#### 项目集的闭包（CLOSURE）

```
CLOSURE(I):
  初始: I 中的所有项目
  若 A→α·Bβ ∈ CLOSURE(I)，则把 B→·γ 加入（对所有 B 产生式）
  反复直到不变
```

#### GO 函数

```
GO(I, X) = CLOSURE({A→αX·β | A→α·Xβ ∈ I})
```

#### LR(0) 分析表构造

1. `A→α·Xβ ∈ Iᵢ`，`GO(Iᵢ,X)=Iⱼ`：
   - `X ∈ VT`：`ACTION[i,X] = sj`
   - `X ∈ VN`：`GOTO[i,X] = j`
2. `A→α· ∈ Iᵢ`（A为第j个产生式）：对所有 `a ∈ VT∪{#}`，`ACTION[i,a] = rj`
3. `S'→S· ∈ Iᵢ`：`ACTION[i,#] = acc`
4. 其余填 `ERR`

#### LR(0) 限制（冲突）

- **移近-归约冲突**：同一项目集中既有移近项目又有归约项目 → LR(0) 不允许
- **归约-归约冲突**：同一项目集中有多个归约项目 → LR(0) 不允许

### 5.4 SLR(1) 分析

**改进**：对归约项目 `A→α· ∈ Iᵢ`，只在 `a ∈ FOLLOW(A)` 时归约：

```
ACTION[i, a] = rj，仅当 a ∈ FOLLOW(A)
```

**SLR(1) 冲突消解条件**：

对 `Iᵢ` 中所有归约项目 `A→α·`，其 `FOLLOW(A)` 与移近项目的 lookahead 及其他归约项目的 FOLLOW 均不相交。

**SLR(1) 局限**：FOLLOW 集不考虑上下文，可能归约到"死前缀"（非活前缀）。

### 5.5 LR(1) 分析

**LR(1) 项目**：`[A→α·β, a]`，其中 `a` 为**向前看符号（lookahead）**。

- `[A→α·, a]`：仅当当前输入为 `a` 时执行归约。
- Action 表根据 lookahead `a` 精确决定移进/归约，消除了 SLR(1) 的 FOLLOW 集近似问题。

**初始项目**：`[S'→·S, #]`

**CLOSURE 扩展**：若 `[A→α·Bβ, a] ∈ I`，对每个 `B→·γ`，将 `[B→·γ, b]` 加入（其中 `b ∈ FIRST(βa)`）。

### 5.6 LALR(1) 分析

**合并同心项目集**：将 LR(1) 中项目集核心（忽略 lookahead）相同的项目集合并，lookahead 取并集。

| 特性 | 描述 |
|------|------|
| 表尺寸 | 与 LR(0)/SLR(1) 相当（远小于 LR(1)） |
| 能力 | 强于 SLR(1)，弱于 LR(1) |
| 风险 | 合并可能引入新的**归约-归约冲突**（虽少见） |
| 应用 | yacc/bison 默认使用 LALR(1) |

**能力排序（强→弱）**：

```
LR(1) > LALR(1) > SLR(1) > LR(0)
```

### 5.7 算符优先分析

- **限制**：产生式右部不允许出现两个相邻非终结符。
- **优先关系**（终结符间）：
  - `a = b`：`U→…ab…` 或 `U→…aAb…`
  - `a < b`：有 `U→…aA…` 且 `A ⇒⁺ b…` 或 `A ⇒⁺ Bb`
  - `a > b`：有 `U→…Ab…` 且 `A ⇒⁺ …a` 或 `A ⇒⁺ …aB`
- 每对终结符至多有一种关系 → 算符优先文法。

### 5.8 LR 分析过程示例

文法：`S→AA`, `A→aA|b`，输入 `bab`：

| 栈 | 输入 | 动作 |
|----|------|------|
| `0` | `bab#` | `action(0,b)=s4` |
| `04` | `ab#` | `action(4,a)=r3`（A→b） |
| `02` | `ab#` | `action(2,a)=s3` |
| `023` | `b#` | `action(3,b)=s4` |
| `0234` | `#` | `action(4,#)=r3` |
| `0236` | `#` | `action(6,#)=r2`（A→aA） |
| `025` | `#` | `action(5,#)=r1`（S→AA） |
| `01` | `#` | `action(1,#)=acc` |

## 六、语义分析与语法制导翻译

### 6.1 语义分析任务

- **说明语句**：符号表的填写和查找
- **可执行语句**：生成中间指令代码
- **语义检查**：类型、运算、维数、越界等
- **翻译**：变量存储分配、表达式求值

### 6.2 属性文法

- **属性文法**：带语义属性及计算规则的 CFG，称为属性文法。
- 每个文法符号有若干**语义属性**，如 `E.val`（值）、`E.type`（类型）、`E.place`（存储位置）。

#### 属性分类

| 类型 | 描述 | 计算时机 |
|------|------|---------|
| **综合属性（Synthesized）** | 依赖孩子节点的属性和自身继承属性 | 归约时（自底向上） |
| **继承属性（Inherited）** | 依赖产生式左部符号或左侧兄弟符号的属性 | 派生时（自顶向下） |
| **固有属性（Inherent）** | 用户给定的值（词法属性，如常量类型） | 归类为综合属性 |

- **S属性文法**：仅含综合属性，可直接在自底向上分析中计算。
- **L属性文法**：每个 Xi 的继承属性只依赖 X₁…Xᵢ₋₁ 的属性和 A 的继承属性；综合属性依赖右部任意属性。S属性文法是特殊的L属性文法。

**注意**：属性计算规则设计需**避免环状依赖**。

### 6.3 语法制导翻译

- **语法制导定义**：在每条产生式上附加语义子程序（动作），分析时同步执行。
- **自底向上**：在归约时执行语义动作（综合属性，用 `$$`, `$1`, `$2`… 引用）。
- **自顶向下**：在派生时执行语义动作（需前移继承属性的计算）→ **翻译模式（Translation Scheme）**。

**说明语句翻译模式**示例（L属性）：
```
D → T {L.in := T.type} L
T → int  {T.type := integer}
T → real {T.type := real}
L → {L₁.in := L.in} L₁, id {addtype(id.entry, L.in)}
L → id   {addtype(id.entry, L.in)}
```

### 6.4 中间代码形式

#### 逆波兰式（后缀表达式）

| 中缀 | 后缀 |
|------|------|
| `A+B` | `AB+` |
| `A+B*C` | `ABC*+` |
| `(A+B)*(C+D)` | `AB+CD+*` |

- **控制转移**：`p BR`（无条件跳到 POST[p]）；`e p BZ`（e 为假跳到 POST[p]）
- `if e then s1 else s2`：`e' p1 BZ s1' p2 BR s2'`

#### 四元式（Three-Address Code）

格式：`(op, arg1, arg2, result)` 等价于 `result := arg1 op arg2`

示例（`A + B*C`）：
```
(*, B, C, T₁)
(+, A, T₁, T₂)
```

特殊格式：
- 一元运算：`(op, arg1, -, result)`
- 无条件转移：`(j, -, -, p)`

**辅助函数**：

| 函数 | 功能 |
|------|------|
| `LookUp(Name)` | 查符号表，返回序号或0 |
| `Enter(Name)` | 向符号表登录新项 |
| `Entry(Name)` | 查/填符号表（优先查） |
| `NewTemp()` | 产生新临时变量，返回编号 |
| `GEN(Op,Arg1,Arg2,Result)` | 产生四元式，NXQ++ |

### 6.5 简单算术表达式翻译

```
AssignSt → Variable := Expr   { GEN(:=, $3.PLACE, 0, $1.PLACE); }
Expr → Expr + Term             { $$.PLACE=NewTemp(); GEN(+,$1.PLACE,$3.PLACE,$$.PLACE); }
Term → Term * Factor           { $$.PLACE=NewTemp(); GEN(*,$1.PLACE,$3.PLACE,$$.PLACE); }
Factor → (Expr)                { $$.PLACE=$2.PLACE; }
       | identifier            { $$.PLACE=Entry($1); }
Variable → identifier          { $$.PLACE=Entry($1); }
```

**类型转换**（混合运算）：定义运算符 `itr`，四元式 `(itr, A, 0, T)` 将整型 A 转为实型 T。

### 6.6 布尔表达式翻译

#### 运算符优先级（高→低）

`(` > `*`, `/` > `+`, `-` > `<`, `<=`, `=`, `>`, `>=`, `<>` > `¬` > `∧` > `∨`

#### 短路估值

- `A∧B`：若 A 为假，不计算 B
- `A∨B`：若 A 为真，不计算 B
- 等价形式：`A∧B` ↔ `(A)?B:0`；`A∨B` ↔ `(A)?1:B`

#### 相关四元式

| 四元式 | 语义 |
|--------|------|
| `(jnz, A, -, p)` | A 为真（非零）时跳转到 p |
| `(jrop, A1, A2, p)` | A1 rop A2 成立时跳转到 p |
| `(j, -, -, p)` | 无条件跳转到 p |

#### 真假出口链（TC/FC）与回填（BackPatch）

- **TC 链（真出口链）**：布尔表达式为真时跳转的四元式序号链
- **FC 链（假出口链）**：布尔表达式为假时跳转的四元式序号链
- **拉链**：多个待填目标的四元式用链表连接，用 `Result` 域存下一节点序号，0 表示链尾
- **`Merge(p1, p2)`**：合并两条链，返回新链头
- **`BackPatch(p, t)`**：用目标地址 t 回填以 p 为首的链

```
E = A∨B<C 的 TC、FC 链：
(1) (jnz, A, -, 0)   ← FC→
(2) (j, -, -, 3)
(3) (j<, B, C, 1)    ← E.TC→
(4) (j, -, -, 0)     ← E.FC→
```

### 6.7 控制语句翻译

#### `if C then S1`

```
         ┌─ C.code ─┐
    C.true:→ S1.code
    C.false:→ S.next
```

#### `if C then S1 else S2`

```
    C.code
    C.true: → S1.code
              goto S.next
    C.false: → S2.code
    S.next:
```

#### `while C do S1`

```
    S.begin:
        C.code
        C.true: → S1.code
                  goto S.begin
        C.false: → S.next
```

示例（`while x>y do z:=x+1`）：
```
L1: if x>y goto L2       ← C.true=S1.begin
    goto Lnext           ← C.false=S.next
L2: t1=x+1
    z=t1
    goto L1              ← S.begin
Lnext:
```

## 七、代码优化与 LLVM IR（第16周）

### 7.1 LLVM IR 基础

- **SSA（静态单赋值）**：每个变量只赋值一次，控制流交汇处用 `phi` 节点。
- **基本块（Basic Block）**：只有一个入口（首条指令）和一个出口（末条指令）的连续指令序列。

**leader 规则**（确定基本块起始）：
1. 程序第一条指令
2. 跳转指令的目标指令
3. 紧跟在跳转指令后面的指令

**控制流图（CFG）**：基本块为节点，控制流为边。用于数据流分析、循环识别、优化。

**支配树（Dominator Tree）**：
- 节点 d **支配** 节点 n：从入口到 n 的所有路径均经过 d
- 用于 SSA 构建（支配边界）、循环识别（回边）

### 7.2 常见 LLVM IR 指令

```llvm
%result = add i32 %a, %b          ; 整数加法
%ptr = getelementptr i32, i32* %base, i32 %idx  ; 数组寻址
%cond = icmp slt i32 %x, %y       ; 有符号小于比较
br i1 %cond, label %true, label %false  ; 条件分支
%v = load i32, i32* %ptr, align 4 ; 从内存读
store i32 42, i32* %ptr, align 4  ; 写入内存
%x = phi i32 [1, %if_true], [2, %if_false]  ; phi 节点
```

### 7.3 常见优化技术

| 优化 | 描述 |
|------|------|
| **常量折叠（Constant Folding）** | 编译期计算常量表达式，如 `5*2=10` |
| **内存提升（Mem2Reg）** | 消除 alloca/load/store，变量提升为 SSA 寄存器 |
| **指令组合（InstCombine）** | `(a+5)*2` → `a*2+10` |
| **强度削弱（Strength Reduction）** | `mul` → `shl`；乘法→加法/移位 |
| **死代码消除（DCE）** | 删除无用代码 |
| **公共子表达式消除（CSE）** | 重复计算 → 复用 |

### 7.4 循环优化

| 优化 | 描述 |
|------|------|
| **循环不变量外提（LICM）** | 将循环体中不变计算移到循环外 |
| **强度削弱 + 归纳变量** | 数组索引 `i*sizeof` → 指针递增 |
| **循环展开（Loop Unrolling）** | 展开 n 倍，减少分支跳转 |
| **循环交换（Loop Interchange）** | 改变嵌套循环顺序，改善 cache 局部性 |
| **循环合并/分裂（Fusion/Distribution）** | 多个循环合并 or 分裂 |
| **自动向量化（Auto-Vectorization）** | 标量 → SIMD 操作（要求无循环携带依赖） |

**循环交换示例（矩阵乘法）**：
```c
// cache差（B跨步访问）     // cache好（B连续访问）
for i for j for k           for i for k for j
  C[i][j]+=A[i][k]*B[k][j]   C[i][j]+=A[i][k]*B[k][j]
```

### 9.1 形式语言基础（推断为第2周内容）

- **文法四元组**：`G = (VN, VT, P, S)`
  - `VN`：非终结符集；`VT`：终结符集；`P`：产生式集；`S`：开始符号
- **Chomsky 文法分类**：
  - 0型：无限制文法
  - 1型：上下文相关（`αAβ → αγβ`，`|γ| ≥ 1`）
  - 2型：上下文无关（`A → γ`，A为单个非终结符）
  - 3型：正则文法（右线性：`A → aB` 或 `A → a`）
- **推导**：`α ⇒ β`（直接推导）；`⇒*`（0步或多步）；`⇒+`（1步或多步）
- **语言**：`L(G) = {x | x ∈ VT*, S ⇒* x}`

### 9.2 文法的二义性（推断为第3周内容）

- **二义性文法**：存在某个句子有两棵或以上不同的语法树。
- 二义性文法不能保证 LR/LL 分析的唯一性 → 一般需消除。
- 但有时用优先级规则处理二义性（如 if-else 悬空）。
