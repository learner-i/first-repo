## 第一部分：概述

### 1.1 编译器的基本概念 ★★★

**编译器（Compiler）**：将某种语言（源语言）编写的程序翻译为另一种语言（目标语言）的程序。

```
源程序 → [编译器] → 目标程序
```

与**解释器（Interpreter）**的区别：

| 比较项 | 编译器 | 解释器 |
|--------|--------|--------|
| 处理方式 | 整体翻译后执行 | 逐条翻译并立即执行 |
| 执行效率 | 高（一次翻译，多次执行） | 低（每次执行都需翻译） |
| 错误报告 | 编译时报告所有错误 | 运行到出错处才报告 |
| 典型代表 | C、C++、Go | Python（CPython）、早期 BASIC |

---

### 1.2 编译器的六个阶段 ★★★

```
源程序（字符流）
      ↓
┌─────────────────┐
│   词法分析       │  → 词法单元流（Token Stream）
└─────────────────┘
      ↓
┌─────────────────┐
│   语法分析       │  → 语法树（Parse Tree / AST）
└─────────────────┘
      ↓
┌─────────────────┐
│   语义分析       │  → 带类型注释的语法树
└─────────────────┘
      ↓
┌─────────────────┐
│ 中间代码生成     │  → 中间表示（如三地址码）
└─────────────────┘
      ↓
┌─────────────────┐
│   代码优化       │  → 优化后的中间表示
└─────────────────┘
      ↓
┌─────────────────┐
│   目标代码生成   │  → 目标机器代码
└─────────────────┘
```

各阶段功能总结：

| 阶段 | 输入 | 输出 | 主要工作 |
|------|------|------|----------|
| 词法分析 | 字符流 | Token 流 | 识别词素，产生 Token |
| 语法分析 | Token 流 | 语法树 | 按文法建立语法树 |
| 语义分析 | 语法树 | 注释语法树 | 类型检查、语义一致性 |
| 中间代码生成 | 注释语法树 | 中间表示 | 生成三地址码等 |
| 代码优化 | 中间表示 | 优化中间表示 | 删除冗余、循环优化等 |
| 目标代码生成 | 优化中间表示 | 目标代码 | 寄存器分配、指令选择 |

**符号表（Symbol Table）**：贯穿所有阶段，存储标识符的名字、类型、作用域、内存位置等属性。

---

### 1.3 两大阶段的划分 ★★

- **分析阶段（Analysis / Front End）**：词法分析 + 语法分析 + 语义分析
  - 与源语言相关，与目标机器无关
  - 检查源程序的正确性
  - 建立中间表示

- **综合阶段（Synthesis / Back End）**：中间代码生成 + 代码优化 + 目标代码生成
  - 与目标机器相关
  - 构造目标程序

> **考试提示**：经常考"某功能属于哪个阶段"，要熟记各阶段的输入输出。

## 第二部分：词法分析

### 2.1 基本概念 ★★★

**词法单元（Token）**：`<token-name, attribute-value>` 二元组
- token-name：词法类别（如 `id`、`number`、`+`）
- attribute-value：附加属性（如指向符号表的指针、数值）

**词素（Lexeme）**：源程序中与 token 模式匹配的实际字符序列

**模式（Pattern）**：描述一类词素形式的规则（用正则表达式描述）

**经典例子**：

```
源程序：position = initial + rate * 60
```

| 词素 | Token |
|------|-------|
| `position` | `<id, ptr→position>` |
| `=` | `<assign>` |
| `initial` | `<id, ptr→initial>` |
| `+` | `<plus>` |
| `rate` | `<id, ptr→rate>` |
| `*` | `<times>` |
| `60` | `<number, 60>` |

**词法分析中的两个实用原则**：
1. **最长匹配原则（Maximal Munch）**：总是选择能匹配的最长词素
2. **优先级原则**：关键字优先于标识符（预置于符号表中）

---

### 2.2 正则表达式 ★★★

#### 基本定义

设 Σ 为字母表，正则表达式（RE）及其所表示的语言 L(r) 递归定义如下：

| 正则表达式 | 语言 | 说明 |
|-----------|------|------|
| `ε` | `{ε}` | 空串 |
| `a`（a ∈ Σ） | `{a}` | 单字符 |
| `r\|s` | `L(r) ∪ L(s)` | 并（选择） |
| `rs` | `L(r)L(s)` | 连接 |
| `r*` | `L(r)*` | Kleene 闭包（零次或多次） |
| `(r)` | `L(r)` | 括号（改变优先级） |

**运算符优先级（从高到低）**：`*` > 连接 > `|`

**常用扩展**：

| 扩展符号 | 等价形式 | 含义 |
|---------|---------|------|
| `r+` | `rr*` | 一次或多次 |
| `r?` | `r\|ε` | 零次或一次（可选） |
| `[a-z]` | `a\|b\|...\|z` | 字符类 |
| `[^abc]` | 非 a、b、c 的字符 | 取反字符类 |
| `.` | 除换行外任意字符 | 通配符 |

#### 代数定律

| 定律 | 说明 |
|------|------|
| `r\|s = s\|r` | 并的交换律 |
| `r\|(s\|t) = (r\|s)\|t` | 并的结合律 |
| `(rs)t = r(st)` | 连接的结合律 |
| `r(s\|t) = rs\|rt` | 分配律 |
| `εr = rε = r` | 连接的单位元 |
| `(r*)* = r*` | 幂等律 |
| `r+ = rr* = r*r` | 正闭包 |
| `r? = r\|ε` | 可选 |

#### 正则定义（Regular Definition）

为正则表达式命名，便于复用：

```
digit   →  0 | 1 | 2 | ... | 9
letter  →  A | B | ... | Z | a | b | ... | z | _
id      →  letter (letter | digit)*
digits  →  digit+
number  →  digits (. digits)? (E [+-]? digits)?
```

### 2.3 有穷自动机 ★★★

#### 2.3.1 NFA（非确定有穷自动机）

**定义**：五元组 `M = (Q, Σ, δ, q₀, F)`

| 符号 | 含义 |
|------|------|
| Q | 有穷状态集合 |
| Σ | 输入字母表 |
| δ | 转移函数：`Q × (Σ ∪ {ε}) → 2^Q` |
| q₀ | 初始状态，q₀ ∈ Q |
| F | 接受状态集合，F ⊆ Q |

**特点**：
- 同一状态对同一输入可转移到多个状态
- 允许 ε 转移（不消耗输入）
- 接受：存在某条路径从 q₀ 到达 F 中某状态

#### 2.3.2 DFA（确定有穷自动机）

**定义**：五元组 `M = (Q, Σ, δ, q₀, F)`

| 符号 | 含义 |
|------|------|
| δ | 转移函数：`Q × Σ → Q`（确定，无 ε 转移） |

**特点**：
- 每个状态对每个输入符号恰好一个转移
- 无 ε 转移
- 实现简单，执行效率高

**NFA 与 DFA 的等价性**：对任意 NFA，存在等价的 DFA（接受相同语言），但 DFA 状态数最多为 NFA 状态数的指数（2^n）。

---

### 2.4 Thompson 构造法（正则表达式 → NFA） ★★★

**基本构造**：

```
ε：         i --ε--> f

a：         i --a--> f
```

**并（r | s）**：

```
              ε→ N(r) →ε
i --ε→ [新初态]          [新终态] --ε→ f
              ε→ N(s) →ε
```

**连接（rs）**：将 N(r) 的终态与 N(s) 的初态合并（或用 ε 连接）

**Kleene 闭包（r*）**：

```
i --ε→ [新初态] --ε→ N(r) --ε→ [新终态] --ε→ f
                   ↑_________ε_________|
         且新初态 --ε→ 新终态
```

**例**：构造识别 `(a|b)*abb` 的 NFA

步骤：
1. 构造 `a` 的 NFA：0 --a→ 1
2. 构造 `b` 的 NFA：2 --b→ 3
3. 构造 `a|b` 的 NFA（并）：4 --ε→ 0,2；1,3 --ε→ 5
4. 构造 `(a|b)*` 的 NFA（闭包）：6 --ε→ 4,7；5 --ε→ 4,7
5. 连接 `abb`：接上 7 --a→ 8 --b→ 9 --b→ 10（终态）

---

### 2.5 子集构造法（NFA → DFA） ★★★

**核心操作**：

- **ε-closure(s)**：从状态 s 出发，只经 ε 转移可到达的所有状态集合（含 s 本身）
- **ε-closure(T)**：T 中所有状态的 ε-closure 的并集
- **move(T, a)**：从 T 中各状态出发，经输入 a 转移可到达的所有状态集合

**算法**：

```
初始化：
  Dstates = {ε-closure({q₀})}，标记为未处理
  Dtran = 空

while 存在未处理的状态 T in Dstates:
    标记 T 为已处理
    for 每个输入符号 a ∈ Σ:
        U = ε-closure(move(T, a))
        if U ∉ Dstates:
            将 U 加入 Dstates，标记为未处理
        Dtran[T, a] = U

DFA 接受状态：Dstates 中包含 NFA 某接受状态的子集
```

**例**：NFA 识别 `(a|b)*abb`

设 NFA 状态为 0-10，初态为 0，终态为 10。

| DFA 状态 | NFA 子集 | 经 a | 经 b |
|---------|---------|------|------|
| A | {0,1,2,4,7} | B | C |
| B | {1,2,3,4,6,7,8} | B | D |
| C | {1,2,3,4,6,7} | B | C |
| D | {1,2,3,4,6,7,9} | B | E |
| E | {1,2,3,4,6,7,10} | B | C |

其中 E 包含终态 10，故 E 为 DFA 接受状态。

### 2.6 DFA 最小化 ★★★

**目标**：构造等价的状态数最少的 DFA（最小 DFA 在同构意义下唯一）。

**算法（表填充法 / Hopcroft 算法核心思想）**：

```
初始划分 П：{F, Q\F}（接受状态组和非接受状态组）

repeat:
    П_new = П
    for 每个组 G in П:
        尝试将 G 分裂：
        若 G 中存在状态 p, q，使得对某输入 a，
        δ(p,a) 和 δ(q,a) 属于 П 中不同的组，
        则将 G 按此分裂为子组
    П = П_new
until П 不再变化

每个组选一个代表状态，构造最小 DFA
```

**例**：对上例 DFA 最小化

初始划分：{E}（接受）和 {A, B, C, D}（非接受）

检查 {A, B, C, D}：
- 对输入 b：A→C，B→D，C→C，D→E
- D 在 b 上转到 E（接受组），其余转到非接受组 → 分裂出 {D}
- 新划分：{E}，{D}，{A, B, C}

检查 {A, B, C}：
- 对输入 b：A→C（在 {A,B,C}），B→D（在 {D}），C→C（在 {A,B,C}）
- B 与 A、C 不同 → 分裂出 {B}
- 新划分：{E}，{D}，{B}，{A, C}

检查 {A, C}：A 和 C 在所有输入上转移到同组 → 不分裂

最终最小 DFA：5 个状态（与原 DFA 相同，本例已最小）。

### 2.7 正则表达式 → DFA 直接构造法 ★★

**步骤**：
1. 将正则表达式 r 扩充为 r# （加结束标记 #）
2. 构造正则表达式的**语法树**
3. 计算每个节点的：
   - **nullable**：该节点能否推导出 ε
   - **firstpos**：该节点代表的子表达式中，可能出现在第一个位置的叶节点集合
   - **lastpos**：可能出现在最后位置的叶节点集合
   - **followpos**：对每个位置 i，followpos(i) 是 i 之后可能跟随的位置集合
4. 利用 firstpos 和 followpos 构造 DFA

**计算规则**：

| 节点类型 | nullable | firstpos | lastpos |
|---------|---------|---------|---------|
| 叶节点 ε | true | ∅ | ∅ |
| 叶节点 i（位置 i） | false | {i} | {i} |
| `c1\|c2` | n(c1) ∨ n(c2) | fp(c1) ∪ fp(c2) | lp(c1) ∪ lp(c2) |
| `c1c2` | n(c1) ∧ n(c2) | 若 n(c1) 则 fp(c1)∪fp(c2) 否则 fp(c1) | 若 n(c2) 则 lp(c1)∪lp(c2) 否则 lp(c2) |
| `c1*` | true | fp(c1) | lp(c1) |

**followpos 计算**：
- 对连接节点 `c1c2`：对 lastpos(c1) 中每个位置 i，将 firstpos(c2) 加入 followpos(i)
- 对闭包节点 `c1*`：对 lastpos(c1) 中每个位置 i，将 firstpos(c1) 加入 followpos(i)

## 第三部分：语法分析——自顶向下

### 3.1 上下文无关文法（CFG） ★★★

**定义**：四元组 `G = (V, T, P, S)`

| 符号 | 含义 |
|------|------|
| V | 非终结符集合（变量） |
| T | 终结符集合 |
| P | 产生式集合，形如 `A → α`，A ∈ V，α ∈ (V∪T)* |
| S | 开始符号，S ∈ V |

**推导（Derivation）**：

- `αAβ ⇒ αγβ`（若 A → γ 是产生式）
- `⇒*`：零步或多步推导
- `⇒+`：一步或多步推导

**最左推导（Leftmost Derivation）**：每步替换最左边的非终结符，记作 `⇒lm`

**最右推导（Rightmost Derivation）**：每步替换最右边的非终结符，记作 `⇒rm`

**语法树（Parse Tree）**：
- 根：开始符号
- 内部节点：非终结符
- 叶节点：终结符或 ε
- 每个内部节点的孩子对应某产生式的右部

**二义性文法（Ambiguous Grammar）**：若文法对某句子存在两棵或以上不同的语法树，则文法是二义的。

**消除二义性**：通过引入优先级和结合性规则改写文法。

例：将二义文法 `E → E+E | E*E | (E) | id` 改写为：
```
E  → E + T | T
T  → T * F | F
F  → (E) | id
```
（乘法优先级高于加法，均左结合）

---

### 3.2 FIRST 集合与 FOLLOW 集合 ★★★

#### FIRST 集合

**定义**：FIRST(α) = 从 α 能推导出的所有串的第一个终结符的集合；若 α ⇒* ε，则 ε ∈ FIRST(α)。

**计算规则**（对文法符号 X）：

```
1. 若 X 是终结符：FIRST(X) = {X}

2. 若 X 是非终结符：
   a. 若 X → ε 是产生式：将 ε 加入 FIRST(X)
   b. 若 X → Y₁Y₂...Yₖ 是产生式：
      - 将 FIRST(Y₁) \ {ε} 加入 FIRST(X)
      - 若 ε ∈ FIRST(Y₁)，将 FIRST(Y₂) \ {ε} 加入 FIRST(X)
      - ...
      - 若 ε ∈ FIRST(Y₁) ∩ ... ∩ FIRST(Yₖ)，将 ε 加入 FIRST(X)

3. 对串 α = X₁X₂...Xₙ，类似计算（将各 Xᵢ 视为上述规则的 Yᵢ）
```

#### FOLLOW 集合

**定义**：FOLLOW(A) = 在某些句型中，紧跟在非终结符 A 之后出现的终结符集合（含 $）。

**计算规则**：

```
1. 将 $ 加入 FOLLOW(S)（S 为开始符号）

2. 若 A → αBβ 是产生式：
   将 FIRST(β) \ {ε} 加入 FOLLOW(B)

3. 若 A → αB 是产生式，或 A → αBβ 且 β ⇒* ε：
   将 FOLLOW(A) 加入 FOLLOW(B)

4. 重复步骤2、3直到所有 FOLLOW 集合不再变化
```

**经典例子**：

文法：
```
E  → TE'
E' → +TE' | ε
T  → FT'
T' → *FT' | ε
F  → (E) | id
```

FIRST 集合：

| 符号 | FIRST |
|------|-------|
| E | {(, id} |
| E' | {+, ε} |
| T | {(, id} |
| T' | {*, ε} |
| F | {(, id} |

FOLLOW 集合：

| 符号 | FOLLOW |
|------|--------|
| E | {$, )} |
| E' | {$, )} |
| T | {+, $, )} |
| T' | {+, $, )} |
| F | {*, +, $, )} |

### 3.3 LL(1) 文法 ★★★

**LL(1) 的含义**：
- 第一个 L：从**左**到右扫描输入
- 第二个 L：构造**最左**推导
- 1：向前看 **1** 个输入符号

**LL(1) 文法判断条件**：

对文法 G 中每个非终结符 A 的每两个不同产生式 `A → α` 和 `A → β`，必须同时满足：

```
条件1：FIRST(α) ∩ FIRST(β) = ∅
条件2：若 ε ∈ FIRST(β)，则 FIRST(α) ∩ FOLLOW(A) = ∅
       （α 和 β 互换同理）
```

等价地：SELECT 集合（预测集合）不相交。

**SELECT(A → α) 的定义**：
```
若 ε ∉ FIRST(α)：SELECT(A→α) = FIRST(α)
若 ε ∈ FIRST(α)：SELECT(A→α) = (FIRST(α) \ {ε}) ∪ FOLLOW(A)
```

LL(1) 条件等价于：对同一非终结符的不同产生式，其 SELECT 集合两两不相交。

### 3.4 消除左递归 ★★★

**直接左递归**：形如 `A → Aα | β`（β 不以 A 开头）

**消除方法**：
```
原文法：A → Aα | β
改写为：
  A  → βA'
  A' → αA' | ε
```

**一般形式**（多个候选式）：
```
原文法：A → Aα₁ | Aα₂ | ... | Aαₘ | β₁ | β₂ | ... | βₙ
改写为：
  A  → β₁A' | β₂A' | ... | βₙA'
  A' → α₁A' | α₂A' | ... | αₘA' | ε
```

**例**：
```
原：E → E + T | T
改：E  → TE'
    E' → +TE' | ε
```

**间接左递归的消除**：

```
算法：
  将非终结符排序为 A₁, A₂, ..., Aₙ
  for i = 1 to n:
    for j = 1 to i-1:
      将 Aᵢ → Aⱼγ 中的 Aⱼ 用 Aⱼ 的当前产生式右部替换
    消除 Aᵢ 的直接左递归
```

### 3.5 提取公因子（Left Factoring） ★★★

**问题**：两个产生式有公共前缀，无法用一个符号区分。

**原始形式**：`A → αβ₁ | αβ₂`

**提取后**：
```
A  → αA'
A' → β₁ | β₂
```

**例**：
```
原：stmt → if expr then stmt else stmt
         | if expr then stmt
改：stmt  → if expr then stmt stmt'
    stmt' → else stmt | ε
```

---

### 3.6 预测分析表的构造 ★★★

**算法**：

```
for 每个产生式 A → α:
    for 每个 a ∈ FIRST(α) 且 a ≠ ε:
        将 A → α 加入 M[A, a]
    if ε ∈ FIRST(α):
        for 每个 b ∈ FOLLOW(A)（含 $）:
            将 A → α 加入 M[A, b]
将所有未定义的 M[A, a] 标记为 error
```

**经典预测分析表**（上述算术表达式文法）：

|  | id | + | * | ( | ) | $ |
|--|----|---|---|---|---|---|
| E | E→TE' | | | E→TE' | | |
| E' | | E'→+TE' | | | E'→ε | E'→ε |
| T | T→FT' | | | T→FT' | | |
| T' | | T'→ε | T'→\*FT' | | T'→ε | T'→ε |
| F | F→id | | | F→(E) | | |

---

### 3.7 预测分析器（LL(1) 分析器）的工作原理 ★★★

**数据结构**：
- 输入缓冲区（含 $）
- 分析栈（初始含 $S）
- 预测分析表 M

**分析算法**：

```
初始化：栈 = [$, S]，ip 指向输入第一个 Token
a = 当前输入符号

loop:
    X = 栈顶符号
    if X == $ and a == $:
        接受，退出
    elif X 是终结符 or X == $:
        if X == a:
            弹出 X，ip 前移（a = 下一个 Token）
        else:
            error()
    else:  // X 是非终结符
        if M[X, a] = X → Y₁Y₂...Yₖ:
            弹出 X，将 Yₖ...Y₂Y₁ 依次压栈（Y₁ 在栈顶）
            输出产生式 X → Y₁Y₂...Yₖ
        else:
            error()
```

**例**：分析 `id + id * id $`

| 步骤 | 栈（顶→底） | 输入 | 动作 |
|------|-----------|------|------|
| 1 | E $ | id+id*id$ | E→TE' |
| 2 | T E' $ | id+id*id$ | T→FT' |
| 3 | F T' E' $ | id+id*id$ | F→id |
| 4 | id T' E' $ | id+id*id$ | 匹配 id |
| 5 | T' E' $ | +id*id$ | T'→ε |
| 6 | E' $ | +id*id$ | E'→+TE' |
| 7 | + T E' $ | +id*id$ | 匹配 + |
| 8 | T E' $ | id*id$ | T→FT' |
| ... | ... | ... | ... |
| 最终 | $ | $ | 接受 |

---

### 3.8 递归下降分析器 ★★

**基本思想**：每个非终结符对应一个递归函数。

```python
def E():
    T()
    E_prime()

def E_prime():
    if lookahead == '+':
        match('+')
        T()
        E_prime()
    # else: ε，不做任何事

def T():
    F()
    T_prime()

def T_prime():
    if lookahead == '*':
        match('*')
        F()
        T_prime()

def F():
    if lookahead == '(':
        match('(')
        E()
        match(')')
    elif lookahead == 'id':
        match('id')
    else:
        error()

def match(t):
    if lookahead == t:
        lookahead = next_token()
    else:
        error()
```

## 第四部分：语法分析——自底向上

### 4.1 自底向上分析概述 ★★★

**基本思想**：从输入串出发，通过**规约（Reduction）**逐步将输入串规约为开始符号。

**规约**：推导的逆过程。若 `A → β` 是产生式，则将句型中的 β 替换为 A。

**移进-规约分析（Shift-Reduce Parsing）**：

使用一个栈，两种基本动作：
- **移进（Shift）**：将当前输入符号压栈
- **规约（Reduce）**：将栈顶的某个串（句柄）规约为对应非终结符
- **接受（Accept）**：分析成功
- **报错（Error）**：语法错误

**句柄（Handle）**：在最右句型中，可以被规约的子串（对应某产生式右部），且规约后仍是最右句型。

---

### 4.2 LR 分析 ★★★

**LR(k) 的含义**：
- L：从**左**到右扫描输入
- R：构造**最右**推导的逆（最右规约）
- k：向前看 k 个符号

**LR 分析器的结构**：

```
输入：a₁ a₂ ... aₙ $
栈：s₀ X₁ s₁ X₂ s₂ ... Xₘ sₘ
动作表 ACTION[s, a]
转移表 GOTO[s, A]
```

**LR 分析的四种动作**：

| 动作 | 含义 |
|------|------|
| `shift sⱼ` | 将当前输入符号和状态 sⱼ 压栈，读入下一个符号 |
| `reduce A→β` | 弹出 \|β\| 个符号和状态，查 GOTO 压入新状态 |
| `accept` | 分析成功 |
| `error` | 语法错误 |

**LR 分析算法**：

```
初始化：栈 = [s₀]，ip 指向输入第一个符号 a

loop:
    s = 栈顶状态，a = 当前输入符号
    if ACTION[s, a] = shift sⱼ:
        将 a 和 sⱼ 压栈
        ip 前移
    elif ACTION[s, a] = reduce A → β:
        弹出 2*|β| 个元素（|β| 个符号和 |β| 个状态）
        t = 新栈顶状态
        将 A 和 GOTO[t, A] 压栈
        输出产生式 A → β
    elif ACTION[s, a] = accept:
        接受
    else:
        error()
```

---

### 4.3 LR(0) 项目与规范 LR(0) 项目集 ★★★

**LR(0) 项目**：在产生式右部某位置加一个点 `·`，表示已分析的部分。

例：产生式 `A → XYZ` 产生四个项目：
```
A → · XYZ    （待移进 X）
A → X · YZ   （已移进 X，待移进 Y）
A → XY · Z   （已移进 XY，待移进 Z）
A → XYZ ·    （规约项目，可规约）
```

**增广文法（Augmented Grammar）**：加入新开始符号 S'，产生式 `S' → S`，避免开始符号出现在产生式右部。

**项目集的闭包（CLOSURE）**：

```
CLOSURE(I):
    J = I
    repeat:
        for J 中每个项目 A → α · Bβ:
            for B 的每个产生式 B → γ:
                if B → · γ ∉ J:
                    将 B → · γ 加入 J
    until J 不再变化
    return J
```

**GOTO 函数**：

```
GOTO(I, X) = CLOSURE({A → αX · β | A → α · Xβ ∈ I})
```

**规范 LR(0) 项目集族（Canonical LR(0) Collection）**：

```
C = {CLOSURE({S' → · S})}
repeat:
    for C 中每个项目集 I:
        for 每个文法符号 X:
            if GOTO(I, X) ≠ ∅ 且 GOTO(I, X) ∉ C:
                将 GOTO(I, X) 加入 C
until C 不再变化
```

---

### 4.4 SLR(1) 分析表构造 ★★★

**SLR(1)（Simple LR）分析表构造算法**：

```
设 C = {I₀, I₁, ..., Iₙ} 为规范 LR(0) 项目集族

1. 对每个项目集 Iᵢ：
   a. 若 A → α · aβ ∈ Iᵢ（a 为终结符）且 GOTO(Iᵢ, a) = Iⱼ：
      ACTION[i, a] = shift j
   
   b. 若 A → α · ∈ Iᵢ（A ≠ S'）：
      对 FOLLOW(A) 中每个 a：ACTION[i, a] = reduce A→α
   
   c. 若 S' → S · ∈ Iᵢ：
      ACTION[i, $] = accept

2. 对每个项目集 Iᵢ 和非终结符 A：
   若 GOTO(Iᵢ, A) = Iⱼ：GOTO[i, A] = j

3. 未定义的表项均为 error
```

**SLR(1) 冲突**：
- **移进-规约冲突**：同一表项既有 shift 又有 reduce
- **规约-规约冲突**：同一表项有两个 reduce

若存在冲突，文法不是 SLR(1) 文法。

---

### 4.5 LR(1) 项目与规范 LR(1) 分析 ★★★

**LR(1) 项目**：在 LR(0) 项目基础上增加**向前看符号（lookahead）**。

形式：`[A → α · β, a]`，其中 a 为向前看符号（终结符或 $）。

**含义**：当栈顶对应 α，且下一个输入为 a 时，可以规约 A→αβ（当 β=ε 时）。

**LR(1) CLOSURE**：

```
CLOSURE(I):
    J = I
    repeat:
        for J 中每个项目 [A → α · Bβ, a]:
            for B 的每个产生式 B → γ:
                for 每个 b ∈ FIRST(βa):
                    if [B → · γ, b] ∉ J:
                        将 [B → · γ, b] 加入 J
    until J 不再变化
    return J
```

**LR(1) 分析表构造**：与 SLR(1) 类似，但规约动作更精确：

```
若 [A → α ·, a] ∈ Iᵢ（A ≠ S'）：
    ACTION[i, a] = reduce A→α
（只对向前看符号 a 规约，而非整个 FOLLOW(A)）
```

---

### 4.6 LALR(1) 分析 ★★★

**LALR(1)（Look-Ahead LR）**：合并具有相同**核心（core）**的 LR(1) 项目集。

**核心**：LR(1) 项目集中去掉向前看符号后的 LR(0) 项目集合。

**合并规则**：若两个 LR(1) 项目集 I 和 J 的核心相同，则将它们合并，向前看符号取并集。

**LALR(1) 的优点**：
- 状态数与 SLR(1) 相同（远少于规范 LR(1)）
- 分析能力强于 SLR(1)
- 实用性最强（yacc/bison 使用 LALR(1)）

**LALR(1) 可能引入的冲突**：合并后可能出现**规约-规约冲突**（但不会引入新的移进-规约冲突）。

**分析能力比较**：

```
LL(1) ⊂ SLR(1) ⊂ LALR(1) ⊂ LR(1) ⊂ 无二义CFG ⊂ CFG
```

---

### 4.7 各类 LR 分析方法比较 ★★

| 方法 | 状态数 | 分析能力 | 说明 |
|------|--------|---------|------|
| SLR(1) | 少（与LR(0)相同） | 弱 | 用 FOLLOW 集决定规约 |
| LALR(1) | 少（与LR(0)相同） | 中 | 合并同核心LR(1)状态 |
| LR(1) | 多（可能指数级） | 强 | 精确向前看符号 |

## 第五部分：语义分析与中间代码生成

### 5.1 语义分析概述 ★★

**主要任务**：
1. **类型检查（Type Checking）**：确保运算分量类型与运算符兼容
2. **作用域分析**：确保变量在使用前已声明
3. **控制流检查**：如 break 只能在循环内使用
4. **唯一性检查**：如 case 标签不重复

**语义错误 vs 语法错误**：
- 语法错误：`a + * b`（结构错误）
- 语义错误：`int x; x = "hello";`（类型不匹配）

---

### 5.2 属性文法（Attribute Grammar） ★★★

**综合属性（Synthesized Attribute）**：由子节点属性计算得到，信息从叶到根传递（自底向上）。

**继承属性（Inherited Attribute）**：由父节点或兄弟节点属性计算得到，信息从根到叶传递（自顶向下）。

**S 属性文法**：只含综合属性，可与 LR 分析结合（在规约时计算属性）。

**L 属性文法**：每个继承属性只依赖左侧兄弟的属性和父节点的继承属性，可与 LL 分析结合。

**语法制导定义（SDD）**：将属性规则附加到文法产生式上。

**语法制导翻译方案（SDT）**：将语义动作嵌入产生式右部的特定位置。

**经典例子**（桌上计算器）：

```
产生式              语义规则
L → E n             print(E.val)
E → E₁ + T         E.val = E₁.val + T.val
E → T               E.val = T.val
T → T₁ * F         T.val = T₁.val * F.val
T → F               T.val = F.val
F → ( E )           F.val = E.val
F → digit           F.val = digit.lexval
```

---

### 5.3 类型系统 ★★

**基本类型**：integer、float、char、boolean 等

**类型表达式**：
- 基本类型
- 数组类型：`array(I, T)`（下标范围 I，元素类型 T）
- 记录类型：`record(field₁:T₁, ..., fieldₙ:Tₙ)`
- 函数类型：`T₁ → T₂`（参数类型 T₁，返回类型 T₂）
- 指针类型：`pointer(T)`

**类型等价**：
- **结构等价**：两个类型表达式结构相同
- **名字等价**：两个类型必须是同一个类型名

**类型转换**：
- **隐式转换（Coercion）**：编译器自动插入转换（如 int → float）
- **显式转换（Cast）**：程序员显式指定

**类型检查规则示例**：

```
若 E₁ : integer 且 E₂ : integer，则 E₁ + E₂ : integer
若 E₁ : float 且 E₂ : float，则 E₁ + E₂ : float
若 E₁ : integer 且 E₂ : float，则 E₁ + E₂ : float（隐式转换）
```

---

### 5.4 中间表示（Intermediate Representation） ★★★

#### 三地址码（Three-Address Code）

**基本形式**：`x = y op z`（最多三个地址）

**常见三地址指令**：

| 指令形式 | 说明 |
|---------|------|
| `x = y op z` | 二元运算 |
| `x = op y` | 一元运算（取反、取地址等） |
| `x = y` | 赋值 |
| `goto L` | 无条件跳转 |
| `if x goto L` | 条件跳转 |
| `if x relop y goto L` | 关系运算后跳转 |
| `param x` | 传参 |
| `call p, n` | 调用过程 p，n 个参数 |
| `return y` | 返回 |
| `x = y[i]` | 数组读取 |
| `x[i] = y` | 数组写入 |
| `x = &y` | 取地址 |
| `x = *y` | 指针解引用 |
| `*x = y` | 指针赋值 |

**三地址码的实现**：

1. **四元式（Quadruples）**：`(op, arg1, arg2, result)`
2. **三元式（Triples）**：`(op, arg1, arg2)`，用位置编号引用
3. **间接三元式**：三元式 + 指令序列表

**例**：`a = b * -c + b * -c`

```
t1 = minus c        // t1 = -c
t2 = b * t1         // t2 = b * (-c)
t3 = minus c        // t3 = -c
t4 = b * t3         // t4 = b * (-c)
t5 = t2 + t4        // t5 = t2 + t4
a  = t5             // a = t5
```

---

### 5.5 常见语句的翻译 ★★★

#### 赋值语句

```
x = E 的翻译：
  生成 E 的三地址码，得到 E 的值存在 t 中
  生成 x = t
```

#### 布尔表达式与控制流

**短路求值（Short-Circuit Evaluation）**：
- `E₁ && E₂`：若 E₁ 为假，不计算 E₂
- `E₁ || E₂`：若 E₁为真，不计算 E₂

**翻译方法**：使用**回填（Backpatching）**技术

- **truelist**：需要回填"真出口"的跳转指令列表
- **falselist**：需要回填"假出口"的跳转指令列表

```
E → E₁ || E₂：
    backpatch(E₁.falselist, nextinstr)  // E₁ 为假则计算 E₂
    E.truelist = merge(E₁.truelist, E₂.truelist)
    E.falselist = E₂.falselist

E → E₁ && E₂：
    backpatch(E₁.truelist, nextinstr)   // E₁ 为真则计算 E₂
    E.truelist = E₂.truelist
    E.falselist = merge(E₁.falselist, E₂.falselist)
```

#### if-then-else 语句

```
if (B) S₁ else S₂ 的翻译：

B.true:  S₁ 的代码
         goto next
B.false: S₂ 的代码
next:
```

三地址码：
```
100: if B goto 102
101: goto 105
102: [S₁ 的代码]
104: goto [next]
105: [S₂ 的代码]
```

#### while 循环

```
while (B) S 的翻译：

begin: B 的代码（真则进入循环体，假则跳出）
       S 的代码
       goto begin
next:
```

#### for 循环

```
for (E₁; E₂; E₃) S
等价于：
E₁;
while (E₂) { S; E₃; }
```

#### 数组访问

设数组 A 的基地址为 base，元素大小为 w。

一维数组：`A[i]` 的地址 = `base + i * w`

二维数组（行主序）：`A[i][j]` 的地址 = `base + (i * n₂ + j) * w`

三地址码：
```
t1 = i * w
t2 = base + t1
x = t2[0]      // x = A[i]
```

### 5.6 过程（函数）调用的翻译 ★★

**活动记录（Activation Record）**：函数调用时在栈上分配的存储区，包含：

| 区域 | 内容 |
|------|------|
| 返回值 | 函数返回值 |
| 实参 | 传入的参数 |
| 控制链 | 指向调用者活动记录的指针 |
| 访问链 | 指向外层过程活动记录（静态链） |
| 保存的机器状态 | 返回地址、寄存器值 |
| 局部变量 | 过程内声明的变量 |
| 临时变量 | 编译器生成的临时量 |

**调用序列**（call sequence）：
```
param a₁
param a₂
...
param aₙ
call p, n
```

## 第六部分：代码优化

### 6.1 优化的基本概念 ★★

**优化目标**：在保证程序语义不变的前提下，改进代码以提高运行速度或减少内存占用。

**优化的分类**：

| 分类标准 | 类别 |
|---------|------|
| 优化范围 | 局部优化（基本块内）、全局优化（跨基本块）、过程间优化 |
| 机器相关性 | 机器无关优化、机器相关优化 |

**基本块（Basic Block）**：满足以下条件的最大指令序列：
1. 只能从第一条指令进入
2. 只能从最后一条指令离开（无中间跳转）

**基本块的划分**：

```
算法：
1. 确定首指令（Leader）：
   a. 程序第一条指令
   b. 任何跳转指令的目标指令
   c. 紧跟在跳转指令之后的指令
2. 每个首指令到下一个首指令之前的指令序列构成一个基本块
```

**流图（Flow Graph）**：
- 节点：基本块
- 边：控制流（若基本块 B₁ 可能紧接着执行 B₂，则有边 B₁→B₂）

---

### 6.2 常见优化技术 ★★★

#### 局部优化（基本块内）

**1. 公共子表达式消除（Common Subexpression Elimination）**

若表达式 E 在某点之前已被计算，且 E 中变量的值未改变，则可复用之前的结果。

```
原：t1 = a + b
    t2 = a + b    // 公共子表达式
    t3 = t1 * t2

优化后：t1 = a + b
        t3 = t1 * t1
```

**2. 复制传播（Copy Propagation）**

若有 `x = y`，则后续对 x 的使用可替换为 y（在 x、y 值未改变的范围内）。

```
原：x = y
    z = x + 1
优化：x = y
     z = y + 1    // 用 y 替换 x
```

**3. 常量折叠（Constant Folding）**

编译时计算常量表达式。

```
原：x = 2 + 3
优化：x = 5
```

**4. 死代码消除（Dead Code Elimination）**

删除永远不会被执行到的代码，或计算结果永远不被使用的代码。

```
原：x = 5
    x = 10    // 上一行的 x 赋值是死代码（x 在被使用前被重新赋值）
    y = x
```

#### 循环优化

**5. 代码外提（Loop-Invariant Code Motion）**

将循环不变量（每次迭代结果相同的计算）移到循环外。

```
原：while (i < n) {
        x = a * b;    // 循环不变量
        i++;
    }
优化：x = a * b;
     while (i < n) {
         i++;
     }
```

**6. 强度削减（Strength Reduction）**

用代价较低的运算替代代价较高的运算。

```
原：for i = 1 to n:
        t = i * 4    // 乘法

优化：t = 4
     for i = 1 to n:
         [使用 t]
         t = t + 4   // 加法代替乘法
```

**7. 归纳变量消除（Induction Variable Elimination）**

消除循环中与循环变量保持线性关系的冗余变量。

#### 全局优化

**8. 到达定值分析（Reaching Definitions）**

分析每个程序点，哪些变量的定值（赋值）可以"到达"该点（中间没有被重新赋值）。

**9. 活跃变量分析（Live Variable Analysis）**

分析每个程序点，哪些变量在该点之后还会被使用（活跃的）。用于死代码消除和寄存器分配。

**数据流方程**（活跃变量）：

```
IN[B]  = USE[B] ∪ (OUT[B] \ DEF[B])
OUT[B] = ∪_{S是B的后继} IN[S]
```

其中：
- USE[B]：基本块 B 中，在定值之前被使用的变量集合
- DEF[B]：基本块 B 中，被定值的变量集合（在使用之前）

---

### 6.3 数据流分析框架 ★★

**通用数据流分析框架**：

| 属性 | 到达定值 | 活跃变量 |
|------|---------|---------|
| 方向 | 前向（forward） | 后向（backward） |
| 汇聚运算 | ∪（并集） | ∪（并集） |
| 初始值 | OUT[Entry]=∅ | IN[Exit]=∅ |
| 传递函数 | OUT=GEN∪(IN\KILL) | IN=USE∪(OUT\DEF) |

**迭代算法（前向分析示例）**：

```
初始化：OUT[Entry] = ∅；对所有其他块 B，OUT[B] = ∅

repeat:
    for 每个基本块 B（除 Entry）:
        IN[B] = ∪_{P是B的前驱} OUT[P]
        OUT[B] = GEN[B] ∪ (IN[B] \ KILL[B])
until 所有 OUT[B] 不再变化
```

## 第七部分：目标代码生成

### 7.1 目标代码生成概述 ★★

**主要任务**：
1. **指令选择**：将中间代码映射为目标机器指令
2. **寄存器分配**：决定哪些变量存放在寄存器中
3. **指令调度**：安排指令顺序以提高流水线效率

**目标代码形式**：
- 绝对机器代码：直接可执行
- 可重定位机器代码：需链接
- 汇编代码：需汇编器处理

---

### 7.2 寄存器分配 ★★★

**图着色法（Graph Coloring）**：

1. 构造**冲突图（Interference Graph）**：
   - 节点：每个变量/临时量
   - 边：若两个变量同时活跃（生命期重叠），则有边（不能分配同一寄存器）

2. **图着色**：用 k 种颜色（k = 寄存器数）对图着色，使相邻节点颜色不同。

3. **溢出（Spill）**：若不能用 k 色着色，选择某变量存入内存（溢出），生成 load/store 指令。

**启发式着色算法（Chaitin 算法）**：

```
while 图中有节点:
    if 存在度数 < k 的节点 n:
        将 n 压栈，从图中删除 n 及其边
    else:
        选择一个节点标记为溢出，删除

弹栈着色：
    从栈中依次弹出节点，分配一个与邻居不同的颜色
    若无法着色，则溢出该节点
```

---

### 7.3 指令选择 ★

**树模式匹配（Tree Pattern Matching）**：

将中间表示（树形式）与目标机器指令模板（树模式）匹配，选择代价最小的指令序列。

**CISC vs RISC**：
- CISC（复杂指令集）：指令丰富，一条指令可完成复杂操作
- RISC（精简指令集）：指令简单，操作数必须在寄存器中，load/store 架构

---

### 7.4 运行时存储组织 ★★

**存储区划分**：

```
高地址
┌──────────────┐
│     栈区      │  ← 函数调用（活动记录）
│   (向下增长)  │
├──────────────┤
│              │
│    堆区       │  ← 动态内存分配（malloc/new）
│   (向上增长)  │
├──────────────┤
│  静态数据区   │  ← 全局变量、静态变量
├──────────────┤
│   代码区      │  ← 程序代码
低地址
```

**静态存储分配**：编译时确定所有变量地址（无递归、无动态内存）

**栈式存储分配**：支持递归调用，活动记录在栈上动态分配/释放

**堆式存储分配**：支持动态内存，需要垃圾回收或手动管理

## 常见考试题型与答题技巧

### 题型一：正则表达式相关 ★★★

**常见题目**：
1. 写出某语言的正则表达式
2. 判断某字符串是否属于某正则表达式描述的语言
3. 化简正则表达式

**答题技巧**：
- 先明确字母表和语言的特征
- 利用基本运算符逐步构造
- 注意运算符优先级：`*` > 连接 > `|`

**例题**：写出所有以 `ab` 开头、以 `ba` 结尾的、字母表为 `{a, b}` 的字符串的正则表达式。

```
答：ab(a|b)*ba | abba
（注意 abba 同时满足以 ab 开头和以 ba 结尾，(a|b)* 可以为空）
更简洁：ab(a|b)*ba
（当 (a|b)* = ε 时，得到 abba，满足条件）
```

---

### 题型二：NFA/DFA 构造与转换 ★★★

**常见题目**：
1. 根据正则表达式构造 NFA（Thompson 构造法）
2. 将 NFA 转换为 DFA（子集构造法）
3. 最小化 DFA
4. 判断 DFA 接受的语言

**答题步骤（NFA → DFA）**：

```
步骤1：计算初始状态 = ε-closure({q₀})
步骤2：对每个未处理的 DFA 状态（NFA 子集）T：
       对每个输入符号 a：
           计算 move(T, a)，再求 ε-closure
           若结果不在 DFA 状态表中，加入
步骤3：含 NFA 终态的子集为 DFA 终态
步骤4：画出 DFA 转移图
```

**例题**：将以下 NFA 转为 DFA

NFA 识别 `a*b`：
- 状态：{0, 1}，初态 0，终态 1
- δ(0, a) = {0}，δ(0, b) = {1}，δ(1, a) = δ(1, b) = ∅

```
ε-closure({0}) = {0}，设为 A

A 经 a：move({0}, a) = {0}，ε-closure = {0} = A
A 经 b：move({0}, b) = {1}，ε-closure = {1}，设为 B

B 经 a：move({1}, a) = ∅，ε-closure = ∅，设为 C（死状态）
B 经 b：move({1}, b) = ∅，ε-closure = ∅ = C

C 经 a：C；C 经 b：C

DFA：A→A（a），A→B（b），B→C（a/b），C→C（a/b）
终态：B（含 NFA 终态 1）
```

---

### 题型三：FIRST/FOLLOW 集合计算 ★★★

**答题步骤**：

```
计算 FIRST：
1. 对每个终结符 a：FIRST(a) = {a}
2. 对每个非终结符 A，扫描 A 的所有产生式：
   A → X₁X₂...Xₖ：
   - 加入 FIRST(X₁) \ {ε}
   - 若 ε ∈ FIRST(X₁)，加入 FIRST(X₂) \ {ε}
   - ...以此类推
   - 若所有 Xᵢ 均可推出 ε，加入 ε
3. 重复直到不动点

计算 FOLLOW：
1. FOLLOW(S) 初始化为 {$}
2. 扫描所有产生式 A → αBβ：
   - 将 FIRST(β) \ {ε} 加入 FOLLOW(B)
   - 若 ε ∈ FIRST(β)，将 FOLLOW(A) 加入 FOLLOW(B)
3. 重复直到不动点
```

**常见错误**：
- 忘记 ε 的传播（当 β ⇒* ε 时，FOLLOW(A) 传给 FOLLOW(B)）
- FOLLOW 集合不包含 ε（FOLLOW 集合只含终结符和 $）
- 计算 FIRST(α) 时，α 是一个串，要按顺序处理

---

### 题型四：构造预测分析表 ★★★

**答题步骤**：

```
1. 计算所有非终结符的 FIRST 和 FOLLOW 集合
2. 对每个产生式 A → α：
   a. 对 FIRST(α) 中每个终结符 a（a ≠ ε）：
      M[A, a] = A → α
   b. 若 ε ∈ FIRST(α)：
      对 FOLLOW(A) 中每个符号 b（含 $）：
      M[A, b] = A → α
3. 检查是否有冲突（多个产生式填同一格）
   若有冲突 → 文法不是 LL(1)
```

**判断 LL(1) 的快速方法**：
- 若文法含左递归 → 一定不是 LL(1)
- 若文法含公共前缀（未提取） → 可能不是 LL(1)
- 构造分析表后有冲突 → 不是 LL(1)

---

### 题型五：LR 分析表构造 ★★★

**SLR(1) 分析表构造步骤**：

```
1. 增广文法：加入 S' → S
2. 构造规范 LR(0) 项目集族：
   a. 初始：I₀ = CLOSURE({S' → ·S})
   b. 对每个项目集 I，对每个文法符号 X：
      GOTO(I, X) = CLOSURE({A→αX·β | A→α·Xβ ∈ I})
   c. 收集所有新生成的项目集
3. 构造 ACTION 表和 GOTO 表：
   a. A→α·aβ ∈ Iᵢ，GOTO(Iᵢ,a)=Iⱼ → ACTION[i,a] = shift j
   b. A→α· ∈ Iᵢ，A≠S' → 对 FOLLOW(A) 中每个 a，ACTION[i,a] = reduce A→α
   c. S'→S· ∈ Iᵢ → ACTION[i,$] = accept
   d. GOTO(Iᵢ,A)=Iⱼ（A 为非终结符）→ GOTO[i,A] = j
```

**常见错误**：
- 忘记增广文法
- CLOSURE 计算不完整
- GOTO 函数漏算某些符号

---

### 题型六：三地址码生成 ★★★

**常见题目**：将某段程序翻译为三地址码

**答题技巧**：
- 每个子表达式引入一个临时变量 t₁, t₂, ...
- 按照运算优先级，从内到外翻译
- 条件语句和循环使用标号（L₁, L₂, ...）和跳转指令

**例题**：翻译 `while (a < b) { c = c + a * b; }`

```
L1: if a >= b goto L2    // 条件取反后跳出
    t1 = a * b
    t2 = c + t1
    c = t2
    goto L1
L2:
```

---

### 题型七：数据流分析 ★★

**常见题目**：计算各基本块的 IN/OUT 集合（到达定值或活跃变量）

**活跃变量分析答题步骤**：

```
1. 对每个基本块 B，计算 USE[B] 和 DEF[B]
   USE[B]：在 B 中，在被定值之前被使用的变量
   DEF[B]：在 B 中，在被使用之前被定值的变量

2. 初始化：IN[B] = ∅（对所有 B）

3. 迭代（后向分析）：
   OUT[B] = ∪_{S是B的后继} IN[S]
   IN[B] = USE[B] ∪ (OUT[B] \ DEF[B])
   重复直到不动点
```

### 综合答题建议

1. **审题仔细**：区分 NFA/DFA、LL(1)/LR(1)/LALR(1)、综合属性/继承属性等概念。

2. **步骤清晰**：计算过程要写清楚每一步，特别是 ε-closure、CLOSURE、GOTO 等操作。

3. **验证结果**：
   - FIRST 集合：用一个简单推导验证
   - 预测分析表：用一个输入串走一遍分析过程验证
   - DFA：用几个样例字符串验证接受/拒绝

4. **重点掌握**（必考内容）：
   - ★★★ 正则表达式 ↔ NFA ↔ DFA 的转换
   - ★★★ FIRST/FOLLOW 集合计算
   - ★★★ LL(1) 预测分析表构造与分析过程
   - ★★★ LR(0) 项目集族构造、SLR(1)/LALR(1) 分析表
   - ★★★ 三地址码生成

5. **容易失分的点**：
   - ε-closure 计算遗漏状态
   - FOLLOW 集合传播规则（A→αBβ 且 β⇒*ε 时的传播）
   - LR 分析表中 GOTO 只针对非终结符，ACTION 只针对终结符
   - 消除左递归后新产生式的 FIRST/FOLLOW 要重新计算

---

> **最后提醒**：编译原理的考试题目往往是综合题，例如"给定文法，判断是否为 LL(1)，若不是则改写，然后构造分析表"。建议将各算法的步骤熟记于心，并通过多做例题培养计算的准确性。考试中遇到计算题，务必一步步写清楚，即使最终结果有误，过程分也能拿到。
